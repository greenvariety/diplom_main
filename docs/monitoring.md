# Мониторинг и логи

## Журнал изменений в системе

URL: `/audit-log/` (только superadmin)

Показывает последние 200 записей `AuditLog`:
- Кто из пользователей сделал действие
- Что за действие (создал / изменил / удалил)
- Какой объект
- JSON данных до и после изменения

## Django Debug Toolbar / Ошибки

В режиме `DEBUG = True` (текущий):
- Ошибки Django показываются прямо в браузере с traceback.
- Консоль `runserver` выводит все HTTP-запросы и SQL-ошибки.

## Логи сервера

При запуске `python manage.py runserver` — всё выводится в терминал:
```
[28/Apr/2026 12:00:00] "GET /dashboard/ HTTP/1.1" 200 5423
[28/Apr/2026 12:00:01] "POST /students/add/ HTTP/1.1" 302 0
```

## Feedback-комментарии

URL: `/feedback/` (любой авторизованный пользователь)

Инструмент разработки — позволяет оставить комментарий к любому элементу интерфейса с указанием CSS-селектора и URL страницы. Данные хранятся в `FeedbackComment`.

## Отладка данных

Быстрая очистка БД для тестов: `/dev/reset-db/` (POST, superadmin only).  
Заполнение тестовыми данными: `python manage.py seed_data`.
