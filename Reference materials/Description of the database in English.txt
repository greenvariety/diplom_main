The student management system database consists of 13 tables and is designed to manage data about students, employees and parents in an educational institution.
Faculty — table of faculties or departments. Contains fields id (int, primary key) and name (string). Represents the top level of the educational institution hierarchy.
Group — table of study groups. Contains fields id (int, primary key), name (string), year (int) — enrollment year, faculty_id (int, foreign key to Faculty) and headteacher_id (int, foreign key to Employee) — the homeroom teacher of the group. One employee can be the homeroom teacher of multiple groups.
Student — table of students. Contains fields id (int, primary key), last_name (string), first_name (string), middle_name (string), birth_date (date), phone (string), email (string), photo (image), status (string), group_id (int, foreign key to Group) and faculty_id (int, foreign key to Faculty). The status field has a fixed set of values: under review, pending enrollment, enrolled, pending expulsion, expelled. Faculty assignment is always required — even when the student is an applicant and no group has been assigned yet. Group assignment is optional.
Parent — table of parents and guardians. Contains fields id (int, primary key), last_name (string), first_name (string), middle_name (string), birth_date (date), phone (string), email (string), photo (image). Has no direct relationship with Student — the relationship is implemented through the StudentParent junction table.
StudentParent — junction table implementing a many-to-many relationship between Student and Parent. Contains fields id (int, primary key), student_id (int, foreign key to Student), parent_id (int, foreign key to Parent), relation_type (string) — type of relationship: mother, father or guardian. One student can have multiple parents, one parent can be linked to multiple students.
Employee — universal table of all staff members of the educational institution. Contains fields id (int, primary key), last_name (string), first_name (string), middle_name (string), birth_date (date), phone (string), email (string), photo (image), position_id (int, foreign key to Position). Used for both teaching staff and administrative personnel.
Position — table of employee positions. Contains fields id (int, primary key), name (string), is_teacher (bool) — whether the position is a teaching position. The list of positions is created and managed by the superadmin.
Subject — table of academic subjects. Contains fields id (int, primary key) and name (string).
GroupSubjectEmployee — junction table that defines which specific employee teaches which subject in which group. Contains fields id (int, primary key), group_id (int, foreign key to Group), subject_id (int, foreign key to Subject), employee_id (int, foreign key to Employee).
Document — universal table of documents for all persons in the system. Contains fields id (int, primary key), owner_type (string) — owner type: student, employee or parent, owner_id (int) — owner id, doc_type (string) — document type, file (file) — document file, uploaded_at (date) — upload date. Allows storing any number of documents for any person without changing the database structure.
User — table of user accounts in the system. Contains fields id (int, primary key), username (string), password (string), role (string), employee_id (int, foreign key to Employee). The role field has a fixed set of values: superadmin, admin, teacher. The superadmin creates accounts and assigns roles.
DeleteRequest — table of deletion requests submitted by administrators. Contains fields id (int, primary key), user_id (int, foreign key to User), object_type (string) — object type: Faculty, Group, Student and others, object_id (int) — object id, reason (text) — reason for deletion, status (string) — request status: pending, approved or rejected, created_at (datetime). Administrators cannot delete objects directly — they can only submit a request. The superadmin reviews the list of requests and makes decisions.
AuditLog — universal log of all changes in the system. Contains fields id (int, primary key), user_id (int, foreign key to User), action (string) — action performed: created, updated or deleted, object_type (string), object_id (int), old_data (text) — state of the object before the change in JSON format, new_data (text) — state after the change in JSON format, created_at (datetime). Populated automatically on any data change in the system.
Relationships: Faculty one-to-many Group. Group one-to-many Student. Faculty one-to-many Student. Student many-to-many Parent through StudentParent. Position one-to-many Employee. Employee one-to-many Group through headteacher_id. Group, Subject and Employee each one-to-many GroupSubjectEmployee. User one-to-many AuditLog. User one-to-many DeleteRequest. Document links to Student, Employee or Parent through owner_type and owner_id fields.
Cascade deletion: deleting a Faculty removes all related Groups and all related Students. Deleting a Group removes all related Students. Deletion is performed only by the superadmin after confirmation with a special password.
