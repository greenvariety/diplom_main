## PLAN задача 7 (текущая) выполнена — Расширенные поля организации

**Что сделано:**

- `core/models.py` — в модель `Institution` добавлены три новых поля:
  - `description` (TextField, blank=True) — описание организации (вместо прежнего `notes`).
  - `photo` (ImageField, upload_to='institutions/') — фото организации.
  - `founded_date` (DateField, null=True, blank=True) — дата фактического основания учебного заведения.
  - Поле `created_at` переименовано в «Дата добавления в систему» (auto_now_add, не редактируется).
- Создана и применена миграция `0010_institution_photo_description_founded_date`.
- `core/api_organizations.py`:
  - `_org_data()` теперь возвращает `description`, `photo` (URL или null), `founded_date` (дд.мм.гггг или null).
  - Добавлена функция `_parse_date()` — разбирает дату из строки (форматы дд.мм.гггг и гггг-мм-дд).
  - POST `/api/organizations/` — принимает `description`, `founded_date` (из FormData) и `photo` (файл).
  - PATCH `/api/organizations/{id}/` — принимает те же поля, обновляет только переданные.
- `core/forms.py` — `OrganizationForm` расширен: поля `description`, `photo`, `founded_date` (было только `code`, `name`, `notes`).
- `frontend/src/modals.jsx` — `OrgFormModal` расширен:
  - Добавлены поля: «Дата основания» (тип date), «Фото» (файл, image/*), «Описание» (textarea).
  - Данные отправляются через FormData (multipart/form-data) — обязательно при загрузке файла.
  - При редактировании — если фото уже есть и не выбрано новое, показывается подсказка «Текущее фото сохранено».
  - Нет серых placeholder-примеров ни в одном поле.
- `frontend/src/screens.jsx` — карточка организации:
  - Если у организации есть фото — аватарка показывает фото (вместо кода аббревиатуры).
  - Под названием вместо `created_at` показывается `founded_date` (дата основания), если она задана.
  - Статистика студентов и сотрудников оставлена без изменений.
- Фронтенд пересобран (`npm run build` — успешно, 0 ошибок).
