## Что сделано

### Мигрировал систему с v1 на v2 (мультитенант)

**models.py**
- `Institution`: добавлен FK `owner` → User, убран `unique=True` с code, добавлен `unique_together = [('owner', 'code')]`
- `User`: роль `superadmin` → `owner`, добавлен `is_owner` property, `is_superadmin` теперь alias для `is_owner`, `institution` FK `CASCADE` → `SET_NULL`
- Добавлена модель `SeedPhrase` (OneToOne к User, хранит `phrase_hash`)
- `AuditLog`: добавлен FK `institution`
- Миграция 0008 применена успешно

**utils.py**
- Добавлены `generate_seed_phrase()`, `hash_seed_phrase()`, `verify_seed_phrase()`
- `get_current_institution()` теперь делегирует в `get_institution_from_session()`
- `get_institution_from_session()`: для owner — берёт из `request.session['institution_id']`, для admin/teacher — из `user.institution`
- Добавлен декоратор `@owner_required`, `@superadmin_required` — alias
- `log_action()` принимает `institution` параметр

**forms.py**
- Добавлены: `OwnerRegisterForm`, `RecoverPasswordForm`, `OrganizationForm`
- `INSTITUTION_ROLE_CHOICES` — убрана роль `superadmin`, остались `admin` и `teacher`
- `PlatformSetupForm` — удалён (был лишним в v2)

**views.py** (~полная перезапись)

- `login_view`: после входа owner редиректит на org-list (или auto-switch если 1 орг)
- Все вьюхи используют `_institution_required(request)` — возвращает `(institution, redirect)` для безопасного редиректа на org-list
- Все `is_superadmin` → `is_owner`, `superadmin_required` → `owner_required`

**urls.py**
- Добавлены маршруты: `/register/`, `/register/seed-phrase/`, `/recover/`, `/organizations/...`
- Убран `/setup/`

**Шаблоны**
