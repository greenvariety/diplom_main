# Структура проекта

---

## Корень

| Файл / папка | Зачем |
|---|---|
| `manage.py` | Django CLI — запуск команд: `runserver`, `migrate`, `makemigrations` и т.д. |
| `launcher.py` | Вспомогательный скрипт для `.bat`-файла — активирует venv и стартует сервер |
| `Запустить проект.bat` | Двойной клик → сервер запущен. Для тех кто не хочет открывать терминал |
| `requirements.txt` | Список Python-зависимостей. `pip install -r requirements.txt` ставит всё нужное |
| `db.sqlite3` | Вся база данных в одном файле. В git не коммитится |
| `.env` | Секреты: SECRET_KEY, EMAIL_HOST_USER, EMAIL_HOST_PASSWORD. Никогда не коммитить |
| `.env.example` | Шаблон `.env` с именами переменных, но без значений. Можно коммитить |
| `.gitignore` | Что git игнорирует: venv, .env, db.sqlite3, __pycache__, media и т.д. |
| `CLAUDE.md` | Инструкция для AI-ассистента — правила, архитектура, соглашения по коду |
| `TASK.md` | Задачи от пользователя — AI берёт по одной и зачёркивает после выполнения |
| `PLAN.md` | Резервный список задач, если TASK.md пуст |
| `STRUCTURE.md` | Этот файл — карта папок и файлов |
| `PROJECT_MAP.md` | Подробная техническая документация: API, модели, роли, маршруты |

---

## `config/` — настройки Django-проекта

| Файл | Зачем |
|---|---|
| `settings.py` | Главный конфиг: БД, JWT, CORS, Email, пути к media/static и все прочие настройки |
| `urls.py` | Корневой роутер — подключает `core/urls.py` и отдаёт медиафайлы в dev-режиме |
| `wsgi.py` | Точка входа для продакшен-серверов (Gunicorn, uWSGI) |
| `asgi.py` | Точка входа для асинхронных серверов — не используется |
| `__init__.py` | Делает папку Python-пакетом (пустой файл, нужен Python) |

---

## `core/` — единственное Django-приложение

Вся бизнес-логика здесь.

### Основные файлы

| Файл | Зачем |
|---|---|
| `models.py` | Все модели БД: User, Institution, Faculty, Group, Employee, Student, Parent, Document и др. |
| `urls.py` | Все `/api/...` маршруты + catch-all → serve_frontend |
| `views.py` | Одна вьюха `serve_frontend()` — отдаёт `frontend/dist/index.html` на все не-API запросы |
| `utils.py` | Общие утилиты: `log_action()`, email-хелперы, декораторы ролей (`@admin_required`) |
| `admin.py` | Регистрация моделей в Django Admin |
| `apps.py` | Конфиг приложения `core` |

### API-файлы (один файл = одна сущность)

| Файл | Что обрабатывает |
|---|---|
| `api_auth.py` | Логин, регистрация с email-кодом, восстановление пароля |
| `api_main.py` | Дашборд — счётчики и сводка |
| `api_organizations.py` | Учебные заведения |
| `api_faculties.py` | Факультеты |
| `api_groups.py` | Группы |
| `api_employees.py` | Сотрудники |
| `api_students.py` | Студенты |
| `api_parents.py` | Опекуны |
| `api_subjects.py` | Предметы |
| `api_positions.py` | Должности |
| `api_documents.py` | Документы (загрузка, скачивание, удаление) |
| `api_users.py` | Управление аккаунтами пользователей |
| `api_profile.py` | Профиль текущего пользователя |
| `api_audit.py` | Журнал изменений |
| `api_notes.py` | Вопросы к записям |
| `api_delete_requests.py` | Заявки на удаление |

### Подпапки `core/`

| Папка | Зачем |
|---|---|
| `migrations/` | Авто-сгенерированные Django-миграции. Не редактировать вручную |
| `management/commands/create_superadmin.py` | `python manage.py create_superadmin` — создаёт owner-аккаунт |
| `management/commands/seed_data.py` | `python manage.py seed_data` — заполняет БД тестовыми данными |
| `tests/` | Автотесты через `requests` к REST API |

## `frontend/` — React SPA

### `src/` — исходный код

| Файл | Зачем |
|---|---|
| `main.jsx` | Точка входа React — монтирует приложение в `<div id="root">` |
| `shell.jsx` | Оболочка приложения: топбар, сайдбар, роутинг между экранами |
| `screens.jsx` | Все экраны: списки, детальные карточки, таблицы |
| `modals.jsx` | Все модальные окна: создание, редактирование, подтверждение |
| `auth.jsx` | Экраны входа, регистрации, восстановления пароля |
| `profile.jsx` | Экран "Мои данные" — просмотр профиля без редактирования |
| `data.jsx` | Константы, иконки, справочные данные |
| `api.js` | Все вызовы к Django API — fetch-обёртки с JWT-токеном |
| `utils.jsx` | Вспомогательные хуки и компоненты (toast, пагинация, Field и т.д.) |
| `styles.css` | Все CSS-стили приложения |
| `assets/` | Картинки и иконки, встроенные в сборку |

### Остальное в `frontend/`

| Файл / папка | Зачем |
|---|---|
| `public/favicon.svg` | Иконка вкладки браузера |
| `public/icons.svg` | SVG-спрайт с иконками интерфейса |
| `public/logo.png` | Логотип приложения |
| `dist/` | Собранный билд (`npm run build`). Django отдаёт его как статику |
| `index.html` | HTML-шаблон для Vite |
| `vite.config.js` | Конфиг сборщика Vite (в dev-режиме проксирует API на Django) |
| `eslint.config.js` | Конфиг линтера JavaScript |
| `package.json` | Node-зависимости и npm-скрипты (`dev`, `build`) |
| `node_modules/` | Установленные npm-пакеты. В git не коммитится |

## `media/` — загружаемые файлы пользователей

Создаётся автоматически при первой загрузке. В git не коммитится.

| Подпапка | Что хранит |
|---|---|
| `institutions/` | Фото учебных заведений |
| `employees/` | Фото сотрудников |
| `students/` | Фото студентов |
| `parents/` | Фото опекунов |
| `users/` | Фото профилей пользователей |
| `documents/` | Прикреплённые документы (PDF, изображения и др.) |

---

## `static/` — статика Django Admin

Собирается через `collectstatic`. Нужна только для Django Admin.  
Фронтенд приложения использует `frontend/dist/`, а не эту папку.

---

## `tests/` — временные тест-скрипты

Здесь создаются и после запуска удаляются временные файлы `autotest_<тема>.py`.  
Постоянные тесты живут в `core/tests/`.

---

## `venv/` — виртуальное окружение Python

Установленные Python-пакеты. В git не коммитится.  
Активация: `venv\Scripts\activate`
