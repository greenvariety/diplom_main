# Git Workflow

## Текущее состояние

Проект на ветке `main`. Есть история коммитов с реализованным функционалом.

## Правила коммитов

Формат сообщения:
```
<тип>: <что сделано>

Примеры:
feat: добавить перевод студента в другую группу
fix: исправить фильтрацию групп для преподавателя
refactor: вынести логику удаления в отдельную функцию
docs: обновить базу знаний
```

Типы:
- `feat` — новая функция
- `fix` — исправление ошибки
- `refactor` — рефакторинг без изменения поведения
- `docs` — изменения в документации
- `chore` — зависимости, настройки, скрипты

- `.env`

## Ветки (рекомендация для диплома)

Для дипломного проекта достаточно одной ветки `main`. Если нужно поэкспериментировать:
```bash
git checkout -b feature/название-фичи
# ... работа ...
git checkout main
git merge feature/название-фичи
git branch -d feature/название-фичи
```

## Теги версий

При сдаче дипломной работы пометить версию тегом:
```bash
git tag -a v1.0 -m "Версия для защиты диплома"
```
