# Статус

## Сессия 2026-05-19 - Реальная отправка писем через Gmail

Починена отправка кодов подтверждения на почту. Раньше они выводились только в консоль сервера и никуда не шли.

Теперь Django отправляет реальные письма через Gmail SMTP:
- Создан `.env` в корне проекта с логином и паролем приложения Gmail (не попадает в git - уже был в .gitignore)
- `settings.py` читает `.env` при запуске и автоматически включает SMTP
- Если `.env` нет - письма снова идут в консоль, сервер не падает
- Если письмо не ушло - пользователь видит сообщение об ошибке, код удаляется из базы

Добавлено правило в `CLAUDE.md`: все пароли и секретные ключи хранить только в `.env`.
Добавлен `.env.example` - шаблон с именами переменных без значений.

## Сессия 2026-05-18 - Email-верификация вместо сид-фразы

**Что удалено:**
- Модель `SeedPhrase` и вся связанная логика (словарь BIP39, хэширование)
- Экран показа сид-фразы после регистрации
- Поля 12 слов из модала удаления организации и восстановления пароля

**Регистрация:**
- Пользователь вводит логин, ФИО, email, пароль
- На email приходит 6-значный код (буквы A-Z + цифры 0-9)
- Второй шаг: ввод кода - создаётся аккаунт, выполняется вход

**Удаление организации:**
- Нажать кнопку удаления - кнопка "Отправить код на почту"
- На email владельца приходит код
- Второй шаг: ввести код - организация удаляется

**Восстановление пароля:**
- Шаг 1: ввести логин, нажать "Получить код" - на email приходит код
- Шаг 2: ввести код + новый пароль

**Техническая часть:**
- Новая модель `EmailCode` в БД (миграция 0012)
- Поле `email` добавлено в модель `User`
- Email-бэкенд: выводит письма в консоль Django сервера (для разработки)
- Новые API: `/api/auth/verify-email/`, `/api/auth/recover/send-code/`, `/api/organizations/<id>/send-delete-code/`
- Все 30 тестов прошли успешно
